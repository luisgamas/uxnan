# Desktop — internationalization (i18n)

![Source](https://img.shields.io/badge/source_of_truth-en.ts-2ea44f?style=for-the-badge)
![Type-safe](https://img.shields.io/badge/missing_key-fails_to_compile-CB3837?style=for-the-badge&logo=typescript&logoColor=white)

The UI is multilingual. **English is the source of truth**; Spanish ships too.
The active language follows the device by default and can be changed in
**Settings → Language**.

## How it works

- Message dictionaries live in [`src/lib/i18n/locales/`](../src/lib/i18n/locales/):
  one file per language (`en.ts`, `es.ts`, …). `en.ts` defines the keys and is
  the canonical `MessageKey` type; every other locale is typed
  `Record<MessageKey, string>`, so a **missing key fails to compile**.
- [`src/lib/i18n/index.ts`](../src/lib/i18n/index.ts) exposes the translator:
  - `i18n.t("key", { name })` — translate a key, interpolating `{name}`-style
    placeholders. Unknown keys fall back to English, then to the key itself.
  - `i18n.plural(n, "x.one", "x.other")` — pick singular/plural by `n` and
    interpolate `{n}`.
  - `i18n.locale` — the active locale code, derived from `app.settings.language`
    (`"system"` → the device language via `navigator.language`) or English.
- Calls are **reactive**: `i18n.t(...)` reads `app.settings.language`, so the UI
  re-renders when the language changes (no reload).

## Using it in a component

```svelte
<script lang="ts">
  import { i18n } from "$lib/i18n";
</script>

<span>{i18n.t("sidebar.projects")}</span>
<button title={i18n.t("project.openTerminal", { name: repo.name })}>…</button>
<Badge>{i18n.plural(n, "project.worktreeOne", "project.worktreeOther")}</Badge>
```

When you add a user-facing string, add its key to **`en.ts` first** (and to
every other locale, or it won't compile).

## Adding a language

1. Copy `src/lib/i18n/locales/en.ts` to `src/lib/i18n/locales/<code>.ts`
   (e.g. `fr.ts`), rename the export, and translate every value. Keep the keys
   and `{placeholders}` exactly as in English.
   ```ts
   import type { MessageKey } from "./en";
   export const fr: Record<MessageKey, string> = { /* …translated… */ };
   ```
2. Register it in `src/lib/i18n/index.ts`:
   ```ts
   import { fr } from "./locales/fr";
   const dictionaries = { en, es, fr };
   export const LOCALES = [
     { code: "en", name: "English" },
     { code: "es", name: "Español" },
     { code: "fr", name: "Français" },
   ] as const;
   ```
3. Run `npm run check` — TypeScript will flag any missing or extra keys. Done:
   the language now appears in **Settings → Language** and is auto-selected for
   devices whose locale matches (e.g. `fr-CA`).

## Notes
- The chosen language persists in `AppSettings.language` (`"system"` or a locale
  code) via the backend, like other settings.
- Brand names (e.g. "Uxnan Desktop", "ADE", "Alpha") and git terms (e.g. `main`,
  branch names) are intentionally not translated.
